# 方法论落地检查表

## 🚨 核心原则

### 1. 先调查，后操作
**任何工作进行之前，都必须先调查、分析、研究、查看已有的条件和环境，而不是直接上手进行操作。**

**执行流程**：
1. 调查 - 了解现有配置、环境、依赖
2. 分析 - 评估变更影响、风险点
3. 研究 - 查阅文档、历史配置、最佳实践
4. 确认 - 验证理解正确，制定操作方案
5. 操作 - 执行变更
6. 验证 - 确认变更结果符合预期

### 2. 继承原有环境
- 不直接替换，要增量修改
- 保留原有配置的备份
- 理解原有设计的意图

### 3. 验证结果
- 修改后必须验证
- 使用测试验证功能
- 检查日志确认无错误

### 4. 保留回滚方案
- 永远要有备份
- 记录回滚步骤
- 测试回滚是否可行

## 📋 操作检查清单

### 配置文件修改
- [ ] 现有配置是什么？
- [ ] 需要修改哪些部分？
- [ ] 是否有备份？
- [ ] 修改后如何验证？
- [ ] 如何回滚？

### 服务器操作
- [ ] 当前服务状态是什么？
- [ ] 操作会影响哪些服务？
- [ ] 是否需要停机？
- [ ] 回滚方案是什么？

### 代码变更
- [ ] 是否有测试覆盖？
- [ ] 是否影响现有功能？
- [ ] 是否需要数据库迁移？
- [ ] 部署后如何验证？

## 🔧 常用验证命令

```bash
# 检查现有证书
openssl x509 -in <cert> -noout -dates

# 检查 Nginx 配置
nginx -t

# 验证服务状态
systemctl status <service>

# 检查日志
journalctl -u <service> -f
```
